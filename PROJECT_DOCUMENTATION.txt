PROJECT OVERVIEW
================================================================================
This is a comprehensive Rasa AI ChatBot system with dynamic agent query 
capabilities. The system handles 40+ different query types without hardcoding,
using intelligent intent classification, entity extraction, and query-based
data retrieval.

ARCHITECTURE
================================================================================
Frontend (React) <-> Backend (Node.js/Express) <-> Rasa Core <-> Rasa Actions
                                                      |
                                                 ChromaDB Vector Store

TECHNOLOGY STACK
================================================================================
- Frontend: React 18.2.0, Axios
- Backend: Node.js, Express 4.18.2, CORS
- AI/NLP: Rasa 3.x, Python
- Vector Database: ChromaDB with all-MiniLM-L6-v2 embeddings
- Data Storage: JSON files (agentData.json, agentLogin.json)

CORE FEATURES
================================================================================

1. DYNAMIC QUERY SYSTEM
   - No hardcoded responses
   - All data from JSON files
   - Query-based architecture
   - Easy to add new agents without code changes

2. INTELLIGENT NLP PROCESSING
   - Intent classification (10 different intents)
   - Entity extraction (names, IDs, emails, dates)
   - Smart keyword detection
   - Flexible matching (handles typos and variations)

3. COMPREHENSIVE QUERY TYPES
   - Agent lookup (by name, ID, email)
   - Field queries (email, company, nationality, agent ID)
   - Full profile queries
   - Login analytics (last login, count, history, first login)
   - Company-based queries
   - Nationality-based queries
   - Statistical queries (most/least active, top agents, totals)
   - Date range queries

4. VECTOR DATABASE INTEGRATION
   - ChromaDB for semantic similarity search
   - Better natural language understanding
   - Persistent query pattern storage
   - Handles variations and typos

SUPPORTED INTENTS
================================================================================
- greet: Welcome messages
- goodbye: Farewell messages
- query_agent: Basic agent lookup
- query_field: Specific field queries
- query_profile: Full profile requests
- query_login: Login-related queries
- query_by_company: Company-based queries
- query_by_nationality: Nationality-based queries
- query_statistics: Statistical queries
- query_date_range: Date range queries
- nlu_fallback: Fallback handling

ENTITIES
================================================================================
- query: Main search term (names, IDs, emails, companies)
- start_date: Start date for range queries
- end_date: End date for range queries

INSTALLATION & SETUP
================================================================================

Prerequisites:
- Python 3.8+
- Node.js 16+
- npm or yarn

1. INSTALL DEPENDENCIES

Backend:
cd backend
npm install

Frontend:
cd frontend
npm install

Rasa Actions:
cd rasa/actions
pip install -r requirements.txt

Vector Database:
install-vectordb.bat

2. TRAIN RASA MODEL
train-model.bat

3. START SERVICES

Method 1 - All at once:
START_PROJECT.bat

Method 2 - Individual terminals:
Terminal 1: start-actions.bat
Terminal 2: start-rasa.bat
Terminal 3: cd backend && npm start

4. VERIFY SETUP
python verify_setup.py

5. TEST SYSTEM
python test_queries.py

API ENDPOINTS
================================================================================

Backend API (http://localhost:5000/api/internal/):
- POST /agent/by-name - Find agent by name
- POST /agent/by-id - Find agent by ID
- POST /agent/by-email - Find agent by email
- POST /agent/by-company - Find agents by company
- POST /agent/by-nationality - Find agents by nationality
- POST /agent/nationality - Get agent nationality
- POST /agent/last-login - Get last login info
- POST /agent/login-count - Get login count
- POST /agent/login-history - Get login history
- POST /agent/first-login - Get first login
- POST /agent/full-profile - Get complete profile
- GET /agent/most-active - Get most active agent
- GET /agent/least-active - Get least active agent
- GET /agent/no-logins - Get agents with no logins
- GET /agent/top-by-login - Get top agents by login count
- GET /agent/count - Get total agent count
- GET /login/this-month - Get this month's logins
- POST /login/date-range - Get logins in date range
- POST /login/by-id - Get login by ID

Rasa API (http://localhost:5005/):
- POST /webhooks/rest/webhook - Send message to bot
- GET /model - Get current model info
- POST /model/train - Train new model

DEVELOPMENT WORKFLOW
================================================================================

1. ADDING NEW AGENTS
   - Edit backend/src/data/agentData.json
   - Add agent object with required fields
   - Restart backend server
   - No code changes needed

2. ADDING NEW QUERY TYPES
   - Add training examples to rasa/data/nlu.yml
   - Update rasa/domain.yml with new intent
   - Add conversation flow to rasa/data/stories.yml
   - Update rasa/actions/actions.py with routing logic
   - Add backend API endpoint if needed
   - Retrain model: train-model.bat

3. MODIFYING RESPONSES
   - Update backend/src/services/agentService.js
   - Modify response formatting
   - Restart backend server

4. IMPROVING ACCURACY
   - Add more training examples to nlu.yml
   - Retrain model
   - Test with test_queries.py

TROUBLESHOOTING
================================================================================

Common Issues:

1. Model Not Found
   Solution: Run train-model.bat

2. Action Server Not Responding
   Solution: Check port 5055, restart start-actions.bat

3. Backend API Errors
   Solution: Check port 5000, verify data files exist

4. Intent Not Recognized
   Solution: Add more training examples, retrain model

5. Slow Response Times
   Solution: Check all services running, optimize queries

6. Vector Database Issues
   Solution: Run install-vectordb.bat, check ChromaDB installation

Debug Commands:
- Check Rasa logs: rasa run --debug
- Check action logs: rasa run actions --debug
- Test API directly: curl http://localhost:5000/health
- Verify model: rasa test
